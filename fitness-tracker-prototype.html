資料庫設計

fitness_tracker — 共 4 張資料表，採 Spring Data JPA 自動建表（ddl-auto: update）

資料表關聯： 一場訓練（workout_session）包含多個訓練組（workout_set）
體重紀錄（body_weight）獨立管理
動作庫（exercise）為預設資料，由系統啟動時自動初始化
workout_session 訓練場次
PK id BIGINT AUTO_INCREMENT
workout_date DATE 訓練日期 NOT NULL
body_part VARCHAR 胸/背/腿/肩/手/核心/全身
note VARCHAR 備註（選填）
created_at DATETIME @PrePersist 自動填入
↕ OneToMany（一場 session 有多個 set）
workout_set 訓練動作組
PK id BIGINT AUTO_INCREMENT
FK session_id BIGINT → workout_session.id
exercise_name VARCHAR 動作名稱 NOT NULL
weight_kg DOUBLE 重量（null = 徒手）
sets INT 組數
reps INT 次數
body_weight 體重紀錄
PK id BIGINT AUTO_INCREMENT
weight_kg DOUBLE NOT NULL
recorded_date DATE NOT NULL
time_of_day VARCHAR MORNING / EVENING / OTHER
note VARCHAR 備註（選填）
created_at DATETIME @PrePersist 自動填入
exercise 動作庫（預設資料）
PK id BIGINT AUTO_INCREMENT
name VARCHAR 動作名稱（如：臥推）
body_part VARCHAR 適用部位
ℹ️ 由 ExerciseDataInitializer 在應用啟動時自動寫入
JPA 關聯設定
@OneToMany(cascade = CascadeType.ALL, orphanRemoval = true)
WorkoutSession → WorkoutSet：刪除 session 時自動刪除所有 set
fetch = FetchType.EAGER
查詢 session 時同時載入所有 set（避免 LazyInitializationException）

HTTP 路由清單

MVC 頁面路由 + REST API 路由

MVC 頁面路由（回傳 HTML）
Method 路徑 Controller 說明
GET / HomeController Dashboard：最新體重、本週訓練次數、最近紀錄
GET /workout WorkoutController 顯示訓練紀錄頁面，傳入所有 session 清單
POST /workout WorkoutController 新增訓練（workoutDate + bodyPart + 多個動作）
POST /workout/delete/{id} WorkoutController 刪除整筆訓練（含所有動作，CascadeType.ALL）
GET /body-weight BodyWeightController 顯示體重紀錄頁面，傳入所有紀錄清單
POST /body-weight BodyWeightController 新增體重（weightKg + recordedDate + timeOfDay）
POST /body-weight/delete/{id} BodyWeightController 刪除一筆體重紀錄
REST API 路由（回傳 JSON）
Method 路徑 Controller 說明
GET /api/workouts WorkoutApiController 取得所有訓練紀錄（JSON）
POST /api/workouts WorkoutApiController 新增訓練（JSON body）
GET /api/body-weight BodyWeightApiController 取得所有體重紀錄（JSON）
POST /api/body-weight BodyWeightApiController 新增體重紀錄（JSON body）
GET /api/exercises ExerciseApiController 取得動作庫清單（JSON）

Thymeleaf 模板技術說明

Server-side rendering — 後端填入資料後回傳完整 HTML

使用到的指令
th:text 輸出變數值到 HTML 元素
th:if / th:unless 條件顯示/隱藏
th:each 迴圈渲染清單（for-each）
th:action / th:href 動態產生 URL（含路徑變數）
th:replace 引入共用 fragment（navbar）
資料傳遞方式
// Controller 端
model.addAttribute("sessions", list);

// Thymeleaf 模板
<tr th:each="s : ${sessions}">
  <td th:text="${s.workoutDate}">...</td>
</tr>

本機執行方式

工程師接手時的啟動步驟

環境需求
  • ✅ Java 17+
  • ✅ Maven（或使用 mvnw）
  • ✅ MySQL 8.0+
  • ✅ 建立資料庫 fitness_tracker
啟動指令
# 1. 設定環境變數
DB_USERNAME=root
DB_PASSWORD=yourpassword

# 2. 執行
./mvnw spring-boot:run

# 3. 開啟瀏覽器
http://localhost:8080
注意：資料庫帳密存放於 .env 檔案，由 application.properties 以 ${DB_USERNAME} 讀取，不應 commit 至 Git。 ddl-auto=update 會在啟動時自動建立/更新資料表結構，無需手動執行 SQL。
