數據規則
欄位名稱 資料型態 說明
id INT · PK 主鍵唯一，自動遞增，不可修改
session_id INT · FK 外鍵關聯 workout_session，不可為空
exercise_name VARCHAR 從 exercise 表取得，確保命名一致性
weight_kg DOUBLE 重量精確到小數（如 82.5 kg）
recorded_date DATE 記錄日期，不可為空，格式 YYYY-MM-DD
time_of_day VARCHAR 限制值：MORNING / EVENING / OTHER
preset BOOLEAN true=系統預設不可刪 · false=用戶自訂可刪
created_at DATETIME @PrePersist 自動填入，不接受外部傳入值
數據邏輯
  • 🗑️ 刪除 workout_session 時，底下所有 workout_set 會透過 CascadeType.ALL + orphanRemoval=true 一併刪除，不會留下孤立資料
  • ⭐ App 啟動時 ExerciseDataInitializer 檢查 exercise 表，若為空則自動寫入 50 筆預設動作（胸/背/腿/肩/手臂/核心），之後啟動不重複寫入
  • 🔍 前端動作輸入支援兩種模式：按部位篩選下拉 ?bodyPart=胸，或文字模糊搜尋 ?q=臥推，兩者都由 ExerciseRepository 的方法命名查詢處理，不用寫 SQL
  • 🧮 1RM 公式（Brzycki）：1RM = weight × (36 ÷ (37 − reps))，從 workout_set 讀取最新訓練數據後計算，結果用於建議下次訓練重量（×80% 肌肥大目標）
  • 📊 體重趨勢查詢：findAllByOrderByRecordedDateDescCreatedAtDesc() 依日期降冪排序，最新一筆可透過 GET /api/body-weights/latest 取得，供首頁 Dashboard 顯示
  • 🔒 新增自訂動作時先呼叫 existsByName() 確認名稱不重複，若已存在回傳 409 Conflict，避免 exercise 表出現重複動作名稱
